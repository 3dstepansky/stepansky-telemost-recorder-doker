# Спецификация Docker-контейнера (v0.002+)

Этот документ описывает границы ответственности между Docker-контейнером и внешним оркестратором (n8n).

## 🐋 Что делает Docker-контейнер (Worker)

1.  **Automation (Puppeteer)**:
    - Заходит в Телемост.
    - Обходит лобби, меняет имя бота.
    - Мониторит количество участников (Auto-Exit).
2.  **Audio Capture**:
    - Прямой захват WebRTC аудио-потока.
    - Сохранение в сырой `.webm`.
3.  **Local Audio Processing (FFmpeg)**:
    - **VAD (Voice Activity Detection)**: Удаление тишины сразу после записи (экономит трафик и время ИИ).
    - **Segmentation**: Нарезка файла на чанки по 20 минут (обход лимитов API).
4.  **Signal Delivery**:
    - Вызов Webhook n8n по завершении всех локальных задач.

## 🧠 Что делает n8n (Orchestrator)

1.  **Telegram Bot Flow**:
    - Обработка команд `/start`, `/stop`, `/meeting_join`.
2.  **Persistence**:
    - Сохранение метаданных встречи в Supabase/Postgres.
3.  **AI Orchestration**:
    - Отправка аудио-чанков в **Groq API** (Whisper).
    - Объединение результатов транскрибации.
    - Отправка текста в **OpenRouter** для генерации Sumarry.
4.  **Notifications**:
    - Отправка финального отчета пользователю в Telegram.

## 🤝 Интерфейс взаимодействия
- **Вход**: n8n вызывает `.sh` скрипты на сервере, которые пробрасывают параметры в Docker `run.js`.
- **Выход**: Контейнер вызывает HTTP Webhook n8n по завершении записи.
