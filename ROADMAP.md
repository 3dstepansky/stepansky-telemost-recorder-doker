# Дорожная карта: Telemost Zero-Cost Bot (Milestones)

Этот документ является маркером готовности нашей системы. Мы идем по этапам от локального прототипа до «облачного» Docker-безотказного решения.

---

## 🟢 Майлстоун 1: Локальное Ядро (Ready)
**Маркер**: Бот заходит как гость, пишет аудио и делает расшифровку через Groq.
- [x] Скрипт `recorder.js` с WebRTC-интерцептором.
- [x] Интеграция Whisper (Groq) для расшифровки.
- [x] Базовая суммаризация (Llama-3).

---

## 🟡 Майлстоун 2: Докеризация рекордера
**Маркер**: Команда `docker run` принимает URL и выдает готовый файл в примонтированную папку.
- [ ] Создание **Dockerfile** (образ с Chrome, Puppeteer и всеми Linux-зависимостями).
- [ ] Оптимизация под Headless-режим (запуск без видимого окна).
- [ ] Настройка volume-маппинга для папки `/recordings`.

---

## 🔵 Майлстоун 3: Интеграция Хранилища (Storage)
**Маркер**: Файлы автоматически улетают в S3 (или ваш аккаунт) сразу после записи.
- [ ] Добавление модуля S3 (AWS SDK / Boto3 / rclone).
- [ ] Настройка переменных окружения для бакетов.
- [ ] Автоматическая очистка временных файлов в контейнере.

---

## 🟣 Майлстоун 4: Оркестрация через n8n
**Маркер**: n8n получает ссылку в Telegram и удаленно запускает Docker-контейнер.
- [ ] Создание Webhook-узла в n8n под "Zero Cost".
- [ ] Настройка SSH/Docker команды запуска из n8n.
- [ ] Обратный вызов (Callback): бот сообщает в Telegram, что PDF/Markdown отчет готов.

---

## 🏆 Майлстоун 5: Промышленная стабильность
**Маркер**: Бот сам выходит при тишине или если остался один.
- [ ] Мониторинг количества участников (логика из m-danya).
- [ ] Логирование ошибок в отдельный канал.
- [ ] Поддержка длинных встреч (автоматическое разбиение файлов).
